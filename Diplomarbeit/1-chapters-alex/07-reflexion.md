# Reflexion und Ausblick
## Zusammenfassung der Ergebnisse

Im Rahmen dieser Diplomarbeit wurde das Frontend des Projekts
BIS-Ares aus der Sicht von UI-Design, UX-Konzepten und technischer
Umsetzung betrachtet. Ziel war es, eine benutzerfreundliche und
übersichtliche Weboberfläche zu entwickeln, die den Anforderungen der
Bürgerservice entspricht und gleichzeitig modern, wartbar und
erweiterbar bleibt.

Zu Beginn der Arbeit wurden grundlegende Konzepte aus den Bereichen
User Interface und User Experience behandelt. Dabei zeigte sich, dass
eine gute Benutzeroberfläche nicht nur optisch ansprechend sein muss,
sondern vor allem eine klare Struktur, eine einfache Bedienung und ein
konsistentes Verhalten bieten soll. Gerade bei einer Anwendung wie
BIS-Ares, in der mit vielen Informationen und Formularen gearbeitet
wird, spielt dieser Aspekt eine besonders wichtige Rolle.

Im weiteren Verlauf wurde das Projekt BIS-Ares analysiert und die
Anforderungen an das Frontend näher betrachtet. Dabei wurde deutlich, dass das Frontend
eine zentrale Rolle im Gesamtsystem einnimmt, da es die direkte
Schnittstelle zwischen Benutzer und Anwendung bildet. Aus diesem Grund
musste die Oberfläche so gestaltet werden, dass wichtige Funktionen
schnell erreichbar sind und die Bedienung auch bei komplexeren
Eingaben möglichst einfach bleibt.

Für die technische Umsetzung wurden Angular und Tailwind CSS
verwendet. Angular eignete sich besonders gut für die strukturierte
Entwicklung des Frontends, da das Framework eine komponentenbasierte
Architektur bietet und dadurch eine klare Trennung einzelner Bereiche
ermöglicht. Tailwind CSS unterstützte die Umsetzung eines
einheitlichen und modernen Designs und erleichterte zusätzlich die
Entwicklung responsiver Layouts.

Ein besonderer Schwerpunkt lag auf der Konzeption und Umsetzung
zentraler Bereiche wie Login, Navigation, Dashboard und
Protokoll-Formular. Dabei wurde versucht, theoretische UI- und
UX-Prinzipien direkt in die praktische Umsetzung zu übertragen.
Zusätzlich wurde das Frontend im Rahmen von Tests und einer
anschließenden Evaluation hinsichtlich Funktionalität,
Benutzerfreundlichkeit und technischer Qualität betrachtet.

Zusammenfassend kann festgehalten werden, dass die Entwicklung eines
guten Frontends nicht nur von der Wahl geeigneter Technologien
abhängt, sondern vor allem auch von einer klaren Struktur, einem
durchdachten Designkonzept und der konsequenten Berücksichtigung der
Benutzerperspektive. Die Arbeit an BIS-Ares hat gezeigt, wie wichtig
die Verbindung von technischer Umsetzung und benutzerorientierter
Gestaltung in einem realen Projekt ist.

## Kritische Würdigung

Die größte Herausforderung bleibt die enge Verzahnung von Design- und
Entwicklungsprozessen. Design-Systeme und Komponentenbibliotheken
bieten einen guten Rahmen, erfordern jedoch eine disziplinierte
Anwendung. Der zeitliche Aufwand für Testing und
Performance-Optimierung wird in der Praxis oft unterschätzt. Eine
frühzeitige und iterative Einbindung von Nutzerfeedback ist
entscheidend für den Erfolg, wird aber häufig zugunsten schnellerer
Releases vernachlässigt.

Die im Rahmen dieser Diplomarbeit gewonnenen Erkenntnisse bilden eine
gute Grundlage für die weitere Entwicklung des Systems. Künftig können
zusätzliche Optimierungen, weitere Tests und neue Funktionen ergänzt
werden, ohne die bestehende Grundstruktur grundlegend verändern zu
müssen.

## Zukünftige Arbeiten

Zukünftige Entwicklungen im Frontend-Bereich werden die Arbeit von
UI/UX-Designern und Entwicklern weiter verändern. Angular Signals
(seit Angular 16) bieten eine reaktivere und effizientere
State-Management-Architektur. Die zunehmende Bedeutung von Web
Components und Micro-Frontends wird die Modularisierung von Frontends
weiter vorantreiben. Künstliche Intelligenz wird sowohl im
Design-Prozess (automatische Layout-Generierung, Design-to-Code) als
auch in der Benutzerinteraktion (personalisierte Inhalte, Chatbots)
eine wachsende Rolle spielen.

Für BIS-Ares ergeben sich darüber hinaus konkrete
Erweiterungsmöglichkeiten. Dazu gehören die Integration
fortschrittlicher Analyse- und Berichtsfunktionen, die Optimierung der
mobilen Nutzung sowie die kontinuierliche Verbesserung der
Benutzerführung durch iterative Usability-Tests. Die modulare
Architektur des Frontends bietet hierfür eine solide Basis, die eine
schrittweise Weiterentwicklung ohne grundlegende Umstrukturierung
ermöglicht.

## Lessons Learned – Erkenntnisse aus dem Projekt

Durch die Arbeit am Projekt BIS-Ares habe ich mehrere wichtige
Erfahrungen gesammelt. Besonders deutlich wurde mir, dass eine gute
Planung der Benutzeroberfläche sehr wichtig ist. Wenn man vorher genau
überlegt, welche Bereiche eine Anwendung braucht und wie Benutzer durch
die Anwendung geführt werden sollen, wird die spätere Umsetzung
einfacher.

Eine wichtige Erkenntnis war auch, dass Formulare gut strukturiert sein
müssen. Gerade beim Protokoll-Formular gibt es viele Eingabefelder.
Deshalb ist es wichtig, zusammengehörige Felder logisch zu gruppieren
und Pflichtfelder klar zu kennzeichnen. Dadurch können Benutzer
schneller arbeiten und Fehler bei der Eingabe werden reduziert.

Außerdem habe ich gelernt, wie wichtig eine klare Projektstruktur im
Frontend ist. Durch Angular konnten einzelne Bereiche wie Login,
Dashboard, Navigation und Protokoll-Formular getrennt umgesetzt werden.
Dadurch bleibt der Code übersichtlicher und kann später leichter
erweitert werden.

Ein weiterer wichtiger Punkt war die Zusammenarbeit zwischen Frontend
und Backend. Dabei wurde deutlich, dass Schnittstellen klar definiert
sein müssen. Besonders beim Login ist es wichtig, dass Frontend und
Backend die gleichen Daten erwarten und korrekt miteinander
kommunizieren.

Auch das Testen während der Entwicklung war eine wichtige Erfahrung.
Fehler sollten nicht erst am Ende gesucht werden. Es ist hilfreicher,
einzelne Funktionen direkt nach der Umsetzung zu testen. Dadurch können
Probleme früher erkannt und schneller behoben werden.

Für zukünftige Projekte nehme ich mit, dass technische Umsetzung und
Benutzerfreundlichkeit immer gemeinsam betrachtet werden müssen. Eine
Anwendung soll nicht nur funktionieren, sondern auch verständlich und
angenehm zu bedienen sein. Diese Erfahrung wird mir bei weiteren
Frontend-Projekten helfen.
